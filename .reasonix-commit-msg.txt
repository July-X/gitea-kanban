fix(v0.7.x): onCommentPaste catch 错误链 fallback —— 不再 "未知错误" 占位

用户粘图上传失败时,toast 只显示 "未知错误",无法判断是哪个 throw 路径失败。

根因：onCommentPaste catch 只读 `e2.messageText`,但 uploadPastedImage 内部
的 `throw new Error('未选中项目')` / `throw new Error('文件为空')` /
`throw new Error('未选中合并请求')` / `throw new Error('服务端未返回附件 URL')`
都是 plain Error 实例（不是 ipc-client normalizeError 包过的 UserFacingError），
`e2.messageText` 是 undefined,fallback 到"未知错误"。

修复：catch 处 fallback chain 加 `e2.message || String(e2.cause) || '未知错误'`,
并加 console.error('[onCommentPaste] upload failed:', err) 给 DevTools 一份。
下次再失败用户能看到具体原因（"未选中合并请求" / "服务端未返回附件 URL"
等),不再迷失在"未知错误"占位文案里。

注意：根因是 Wails v2.12 binding 不适合传大 base64（Wails 官方 issue
#3989 已确认 5MB+ 必失败,推荐 Asset Handler 方案 + fetch 上传二进制），
这只能让用户看到具体错误信息,无法彻底解决大图上传问题。本 fix 只是改进
诊断体验,真正的图片上传修复在后续 commit 用 asset handler 重做。

测试：手动粘图（任何大小）触发失败路径,toast 应显示具体 message 而非"未知错误"。

后端 go test ./app/... 全过；前端 typecheck 预先存在的旧错误与本 fix 无关。
