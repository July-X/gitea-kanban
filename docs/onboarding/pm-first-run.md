# PM 第一次上手 Checklist（A-1 准备）

> **目标**：让 PM（**非技术人员**）独立跑通完整使用流程，记录每个步骤的**卡点 / 困惑 / 误操作 / 错误提示看不懂 / 弹窗多余 / 文案不准**。
>
> **用户假设**：
> - 不懂 git 命令行、不懂 gitea 概念
> - **PC-only**：Mac 或 Windows 桌面应用，**不是** Web / 移动端
> - 第一次用 gitea-kanban，但会用 Gitea Web 看过 issue/PR
>
> **PM 操作时我不主动帮**——PM 卡住时只观察，记录痛点；PM 主动问才答。

## 1. 启动应用

| 步骤 | 期望 | 实际 | 痛点 |
|---|---|---|---|
| 双击 app 启动 | 弹出窗口，2 秒内显示登录页 | | |
| 看到登录页 | 明确告诉用户"这是干嘛"+ 下一步操作指引 | | |
| 找到 token 输入框 | 输入框 placeholder 说"在哪里拿 token" | | |
| 输入 token + 点击连接 | 跳到主界面，显示项目列表 | | |
| **如果连不上** | 错误提示说人话（"连不上 gitea，请检查网络 / token 是否过期"） | | |

**重点观察**：
- 启动时间能不能接受？
- 错误提示人话吗？

## 3. 看板视图（核心使用场景）

| 步骤 | 期望 | 实际 | 痛点 |
|---|---|---|---|
| 进入"看板" tab | 看到列（待办 / 进行中 / 已完成） | | |
| 看到卡片 | 每张卡片显示：标题 / 编号 / 负责人 / 状态 | | |
| 点击卡片 | 展开详情（描述 / 评论 / 子任务） | | |
| **拖卡片到另一列** | 列变颜色提示、放下后刷新、显示 toast "已移到 进行中" | | |
| 创建新列 | 输入列名 → 确认 → 列出现 | | |
| 重命名列 | 双击列名 → 输入新名 → 确认 | | |
| 删除列 | 列右键 → 删除 → 二次确认（说"会丢失 N 张卡片"） | | |
| 卡片加载慢 | 列表 > 50 张时滚动卡吗？分页？ | | |

**重点观察**（这是 PM 80% 工作流）：
- 拖卡片流畅吗？（桌面鼠标 + 键盘 ↑↓ 切换焦点）
- "列名 / 卡片名"输入时有最大长度限制吗？超过怎么办？
- 误删列 / 误拖卡片时，有 undo 吗？undo 路径清晰吗？

## 6. 合并请求（Merges）

| 步骤 | 期望 | 实际 | 痛点 |
|---|---|---|---|
| 进入"合并请求" tab | 看到 PR 列表（全部 / 开放 / 已合并 / 已关闭） | | |
| 点行展开 | 看到 PR 详情（作者 / 状态 / 描述 / 文件改动） | | |
| 点"合并"按钮 | 弹二次确认 → 选合并方式 → 确认 → 合并成功 toast | | |
| 合并方式选项 | "普通合并 / 变基 / 变基+合并 / 压缩" — 人话 | | |
| **合并到 main** | 额外警告（"影响所有协作者"） | | |
| 跳 gitea web | 按钮"在 gitea 中打开" — 直接跳浏览器 | | |
| 有冲突 | 按钮 disabled，显示"请先在 gitea 解决冲突" | | |

**重点观察**：
- PM 知道"合并请求"是干嘛的？（vs gitea web 上的 PR）
- 5 种合并方式 PM 看得懂吗？需要简化吗？
- 二次确认弹窗 PM 觉得多余吗？

## 9. 常见 PM 痛点预判（看 PM 实际操作时是否命中）

- [ ] "项目"和"仓库"傻傻分不清
- [ ] "合并请求"和"PR"两个词都出现
