# v0.7.20 —— gh_not_installed 错误码 + 安装引导 toast

> **TL;DR**：Mac 安装版启动后，GitHub 仓库同步报错 `exec: "gh": executable file not found in $PATH`，但错误信息没有引导用户安装 gh。v0.7.20 增加 `gh_not_installed` 错误码（结构化 IpcError），前端捕获后展示"同步失败"toast + "打开安装页"按钮，一键引导用户安装 gh CLI。

## 范围

### 后端（2 个文件）

- `app/ipc/errors.go` —— 新增 `CodeGhNotInstalled = "gh_not_installed"` 常量 + `NewGhNotInstalled(cause)` 构造函数
- `app/git/native.go` —— `CloneWithFilter` / `FetchWithFilter` 的 gh 缺失检查失败时，从 `fmt.Errorf(...)` 改为返回 `*ipc.IpcError{Code: "gh_not_installed"}`（前端可识别）

### 前端（3 个文件）

- `frontend/src/shared/errors.ts` —— `IpcErrorCode.GH_NOT_INSTALLED = 'gh_not_installed'`
- `frontend/src/lib/ipc-client.ts` —— `KNOWN_ERROR_CODES` + `CODE_CATEGORY['gh_not_installed']`（"环境问题"）+ `RECOVERABLE['gh_not_installed']`（true，安装后恢复）
- `frontend/src/lib/github-cli-guide.ts` —— 新增 `buildGhInstallToastError(messageText?, hint?)` helper，返回带 `actions` 按钮的 toast 配置（"打开 GitHub CLI 安装页" → `https://cli.github.com/`）
- `frontend/src/components/StatusBar.vue` —— `onSyncClick` / `onUpdateClick` 的 catch 块加 `if e.code === 'gh_not_installed'` 分支，走 `buildGhInstallToastError`
- `frontend/src/views/TimelineNewView.vue` —— `goToLatest` / `enableGitGraph` 的 catch 块加同样分支

### 测试（2 个文件）

- `app/git/clone_test.go` —— 新增 `TestCloneWithFilter_GhNotInstalled`（临时清 PATH 隔离验证错误码）
- `app/git/sync_test.go` —— 新增 `TestFetchWithFilter_GhNotInstalled`（同样隔离验证）

## 改动详情

### 根因：gh 缺失时错误信息无引导

**症状**：Mac 用户从 .dmg 安装应用后，GitHub 仓库点"同步"或"更新"立即报错：

```
应用出错了：GitHub 仓库走 gh partial fetch 失败: 系统未安装 gh 命令，无法快速加载 GitHub 超大仓库提交记录: exec: "gh": executable file not found in $PATH
```

用户看到这个错误不知道下一步该做什么。

**根因**：`app/git/native.go` 的 `CloneWithFilter` / `FetchWithFilter` 在 `exec.LookPath("gh")` 失败时返回 `fmt.Errorf(...)`，这个普通错误被 Wails ErrorFormatter 序列化后传到前端，前端 `normalizeError` 把它当 "internal" 兜底处理，展示"应用出错了：..."，**没有任何引导动作**。

**修法**：

1. `app/ipc/errors.go` 新增 `CodeGhNotInstalled` 错误码 + `NewGhNotInstalled` 构造函数
2. `app/git/native.go` 的 gh 缺失检查失败时返回 `*ipc.IpcError`（前端 duck-type 识别）
3. 前端 `KNOWN_ERROR_CODES` 加白名单，`CODE_CATEGORY['gh_not_installed'] = '环境问题'`
4. `buildGhInstallToastError` 生成带"打开安装页"按钮的 toast

修复后用户看到的 toast：

```
同步失败：系统未安装 GitHub CLI（gh）
请先安装 gh CLI 后再同步 GitHub 仓库
[打开 GitHub CLI 安装页]
```
