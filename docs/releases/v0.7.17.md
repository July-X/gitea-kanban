### 改动 2 处 CSS

1. **`.pr-detail__event-content` 加 `overflow: hidden`**：

```css
.pr-detail__timeline-item--system .pr-detail__event-content {
  flex: 1;
  min-width: 0;
  display: flex;
  flex-direction: column;
  gap: 2px;
  font-size: var(--font-sm);
  color: var(--color-text-secondary);
  /* v0.7.17：内容超出容器宽度时直接截断（不撑爆 timeline 列宽） */
  overflow: hidden;
}
```

2. **`.pr-detail__event-line` 改 `flex-wrap: nowrap` + 加 `white-space: nowrap` + `overflow: hidden`**：

```css
.pr-detail__event-line {
  display: flex;
  align-items: center;
  gap: 4px;
  flex-wrap: nowrap;        /* v0.7.17：移除 v0.7.10 的 wrap（让长内容换行成 2 行） */
  white-space: nowrap;      /* v0.7.17：主行强制 1 行 */
  overflow: hidden;         /* v0.7.17：溢出截断 */
  font-size: var(--font-body); /* v0.7.10：13px → 14px 字号升一档 */
}
```

### 保留的 block / inline 块行为

- **block 块**（push event commit 列表 `.pr-detail__event-block--commits`）：保留 column 布局 + 每 commit 1 行（对齐 Gitea web `commits_list_small` 模板）
- **inline 块**（v0.7.6 留的 label 兜底单 chip 块 `.pr-detail__event-inline`）：继续 `flex-wrap: wrap` 走窄屏换行（v0.7.14 已把主流情况搬到主行渲染，inline 块只在 label 单 chip 等边缘 case 走）

## 根因

v0.7.10 我加的 `pr-detail__event-line` CSS `flex-wrap: wrap` 让主行内容多时换行。当时是为了支持 author / time 等太长时换行不撑爆容器。但同时把所有事件主行整段换行——user 期望跟 Gitea web 一致 1 行渲染。

v0.7.16 给 merge 事件整段加 `white-space: nowrap` 兜底，但其他事件没改。

## 修复

v0.7.17 直接改主行 CSS：移除 `flex-wrap: wrap`，加 `white-space: nowrap` + `overflow: hidden`。所有事件主行强制 1 行，超出截断（不撑爆容器）。
