## 🚨 红线约束（不可违反）

> **所有开发都不能影响 GitHub 数据源功能。**

具体而言：

1. **GitHub adapter 方法签名不变**：`GitHubAdapter` 已实现的 `VerifyToken` / `CloneRepo` / `LogGraph` 三个方法，返回值与行为不得改变。
2. **GitHub 未实现方法保持 `ErrNotSupported`**：`ListIssues` / `ListLabels` / `ListMembers` / `UpdatePullLabels` / `UpdatePullAssignee` / `UpdatePullReviewers` / `ListPullReviews` / `CreatePullReview` / `ListPullFiles` / `GetPullFileDiff` 等返回 `platform.ErrNotSupported` 的方法，**不得在 v0.6 改为实现**。
3. **PlatformAdapter interface 变更需双端同步**：如果 interface 新增方法（如 `ListMilestones`），GitHub adapter 必须也新增对应方法（可返回 `ErrNotSupported`），保证编译通过。
4. **前端 UI 需 platform-aware**：所有新增 Gitea 功能 UI 必须通过 `platform` 字段判断，GitHub 数据源下隐藏或禁用。
5. **IPC 命名空间不变**：`pulls.*` / `labels.*` / `milestones.*` 等 IPC 通道保持现有命名，新增通道需 v0.6 统一规划。
6. **测试隔离**：新增 Go 测试不得破坏 `github/adapter_test.go` 中现有断言。

**违反红线的 PR 必须修正后才能合入。**
