# v0.7.5 —— 系统事件 UX 文案 + 时间格式对齐 Gitea web

> **TL;DR**：v0.7.4 补了 DisplayName / "评论于" / header actions 等细节，但系统事件行还有 2 处 UX 不对齐 Gitea web：① "事件" 通用回退（4-5 种 type 走 fallback 显示 "kanban_demo 事件"）② 时间独立在右 + 动词缺 "此合并请求" 限定词。v0.7.5 全部修正。

## 定位

接续 v0.7.4 "Timeline 细节补全"，v0.7.5 把系统事件行的**文案**和**时间排版**做对齐：

- **v0.7.4 做过的**（细节）：DisplayName / "评论于" / 系统事件 inline 详情（评审人/合并 SHA/指派人） / 评论 header 右侧 [所有者]/表情/... 菜单
- **v0.7.5 补的**（UX 文案）：
  - systemEventVerb 字典重写，覆盖 22+ 种 Gitea CommentType
  - 移除 v0.7.x "事件" 通用 fallback（不专业）
  - PR 动作加 "此合并请求" 限定词（"关闭了此合并请求" / "置顶了此合并请求"）
  - 时间格式 `X verb  ·  Y 天前` 改成 `X 于 Y verb`（时间融进行内 + "于" 介词）
  - push event 数量解析（"推送了 1 个提交" / "推送了 3 个提交"）

## 范围

### 前端（2 个 commit）

- **`feat(v0.7.5): 系统事件 verb 字典重写 + 时间融进行内 + 移除 '事件' 通用 fallback`**
  - `systemEventVerb(item)` helper 重写：覆盖 22+ 种 Gitea CommentType 全部 case：
    - `close` (2): "关闭了此合并请求" / `reopen` (1): "重新开启了此合并请求"
    - `merge` (28): "合并了提交"（SHA + branch 在 inline）
    - `delete_branch` (11): "删除了分支"（分支名在 inline）
    - `pin` (36): "置顶了此合并请求" / `unpin` (37): "取消了此合并请求的置顶"
    - `lock` (23): "锁定了此合并请求" / `unlock` (24): "解锁了此合并请求"
    - `time tracking` (12-15/26/38): "开始工作" / "完成了计时" / "添加了工作时间" / "取消了计时" / "删除了工作时间" / "修改了时间估算"
    - `project` (30/31): "修改了项目" / "移动到了项目列"
    - `auto merge` (34/35): "已排定自动合并" / "已取消排定自动合并"
    - `ref` (3/5/6/33): 根据 `item.refAction` 区分 "通过引用关闭了" / "通过引用重新开启了" / "交叉引用了" / "引用了"
  - 移除 v0.7.x "事件" 通用 fallback：未识别 type 返回空字符串（不显示 verb）
  - 模板时间格式从 `X verb  ·  Y 天前` 改成 `X 于 Y verb`：
    - 评审事件：`X reviewStateLabel at time` → `X 于 time reviewStateLabel`
    - 系统事件：`X systemEventVerb at time` → `X 于 time systemEventVerb`
  - CSS：`.pr-detail__event-prep` 新增（"于" 介词样式）；`.pr-detail__event-time` 去掉 `margin-left: auto`（不再是右对齐）
  - 删除 unused `systemEventLabel` function（v0.7.5 统一用 `systemEventVerb`）

- **`feat(v0.7.5): push event 数量解析 —— '推送了 N 个提交'`**
  - `systemEventVerb(item)` 处理 `push` (29) type 时正则抠 body 里的 commit 数量
  - body 格式：`"added 1 commit {time}"` / `"added N commits {time}"` / `"添加了 1 个提交 {time}"` / `"添加了 N 个提交 {time}"`（中文 Gitea 渲染）
  - regex：`/(\d+)\s*(commits?|个?提交|个?个提交)/i`
  - `N=1` → `"推送了 1 个提交"`
  - `N>1` → `"推送了 {N} 个提交"`
  - 解析失败 → fallback `"推送了新提交"`
