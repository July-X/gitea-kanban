# v0.7.14 —— label 事件 chip 移到主行

> **TL;DR**：user 反馈 ⑬"修改了标签" 后面 chip 不要单独换一行显示 —— v0.7.14 改 label 事件 chip 渲染从独立 inline 块（`<div pr-detail__event-inline>` 缩进显示）搬到主行（`<div pr-detail__event-line>` 内紧跟 verb span），跟 push / merge 事件同 div 渲染。user 反馈的"push event commit 消息 / delete_branch 分支名还是看不到"v0.7.8 + v0.7.12 master 代码已正确，需要 user 升级 binary 才能看到效果。

## 范围

### 前端（1 个 commit）

- **`feat(v0.7.14): label 事件 chip 移到主行`** —— frontend/src/views/MergesView.vue

## 改动详情

### label 事件 chip 渲染位置

**改前（v0.7.6 改的）**：

```vue
<div class="pr-detail__event-line">
  <span>{{ author }}</span>
  <span>于</span>
  <span>{{ time }}</span>
  <span class="pr-detail__event-verb">{{ verb }}</span>  <!-- "修改了标签" -->
</div>
<!-- 行内附加：独立 div 块 -->
<div v-if="hasSystemEventInlineDetail(item)" class="pr-detail__event-inline">
  <span v-if="item.type === 'label' && (item.addedLabels?.length || item.removedLabels?.length)">
    <span v-for="lbl in item.addedLabels" class="pr-detail__event-label">{{ lbl.name }}</span>
    ...
  </span>
</div>
```

**改后（v0.7.14）**：

```vue
<div class="pr-detail__event-line">
  <span>{{ author }}</span>
  <span>于</span>
  <span>{{ time }}</span>
  <span class="pr-detail__event-verb">{{ verb }}</span>  <!-- "修改了标签" -->
  <!-- v0.7.14：chip 紧跟 verb 后，同一 div 同一行 -->
  <span v-if="item.type === 'label' && (item.addedLabels?.length || ..)" class="pr-detail__event-labels">
    <span v-for="lbl in item.addedLabels" class="pr-detail__event-label">{{ lbl.name }}</span>
    ...
  </span>
</div>
<!-- 行内附加：删 label chip 渲染（已搬到主行），其他类型（milestone/change_target_branch 等）保持 -->
<div v-if="hasSystemEventInlineDetail(item)" class="pr-detail__event-inline">
  <span v-else-if="item.type === 'milestone'">...</span>
  ...
</div>
```

## 影响

实际效果对比（label 事件）：

| 改前 | 改后 |
|---|---|
| 主行: "kanban_bot 于 上个月 修改了标签" | 主行: "kanban_bot 于 上个月 修改了标签 [bug] [feature] [needs-review]" |
| 缩进块: "[bug] [feature] [needs-review]" | （无缩进块） |

对齐 Gitea web 渲染。

## 关于"push event commit 消息 / delete_branch 分支名还是看不到"

user 反馈"还是看不到具体的分支信息" —— 这 2 个 v0.7.8 + v0.7.12 改过：

| 场景 | 改动 | commit |
|---|---|---|
| push event 显示 commit 列表 | block 块用 `item.commitIds` 数组 + `commitDetails(sha)` 从 store `pull.commitsByPR` 缓存按 SHA 短码匹配拿 subject/author | v0.7.8 |
| delete_branch 显示分支名 | verb 直接拼 `\`删除分支 ${item.oldRef.replace(/^refs\/heads\//, '')}\`` | v0.7.12 |

从 user 截图看（"推送了新提交" + "删除了分支" 仍是 v0.7.5 fallback / v0.7.5 verb 字符串）—— **user 跑的还是 v0.7.5 之前的 binary**（master 代码 v0.7.8 + v0.7.12 改了但 binary 没升级）。

升级 v0.7.14 binary 后预期：
- push event "kanban_demo 于 27 天前 推送了 1 个提交" + 缩进 "feat: ... 54244ef2d3"（commit 消息 + 短 SHA）
- delete_branch "kanban_bot 于 21 天前 删除分支 cx-single-891347"（带分支名）
