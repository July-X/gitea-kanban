## 一、规划背景与定位

### 1.1 吃透上下文后的真实 v0.7.0 定位

本规划在修产品文档过程中**重新对齐了产品历史**，纠正了初版的错误假设。

**关键事实**（产品文档已同步更新）：

| 事实 | 来源 |
|---|---|
| v0.5.0 ADR-0008 **计划内**做 GitHub PR 闭环（ListPulls / GetPull / Merge / Close / Comments / Reviews / Files / Diff / Reactions） | `docs/adr/0008-pr-comment-v05-enhancement.md` |
| v0.6.0 **深化** Gitea 功能（Milestones / Review 完整化 / Assignee 多选 / MergesView 三 Tab 重构 / app.go 9 文件拆分） | `b977906` 发版聚合 commit |
| GitHub PR 列表页的 attr-editor（标签 / 指派人 / 评审人 / 里程碑选择器）**前端 v-if 一直挡着**：`currentPlatform === 'gitea'` | `MergesView.vue:477` |
| GitHub adapter 仍有 5 个方法返 `ErrNotSupported` | `app/platform/github/adapter.go` |

**v0.7.0 真正要解决的问题**：

> GitHub 端 PR 属性编辑器**不是"没有实现"，而是"后端数据没有导致前端 v-if 挡着不渲染"**。

前端开了 attr-editor 之后，需要 `labelsList` / `membersList` / `milestonesList` 三个数据源，但 GitHub adapter 这三个都返 `ErrNotSupported`，所以前端即使放开 v-if 也拿不到数据。

### 1.2 v0.7.0 一句话目标

**补全 GitHub 端的 5 个后端方法，使 PR MergesView 的属性编辑器（标签 / 指派人 / 评审人 / 里程碑选择器）在 GitHub 端可用。**

### 1.3 不做

- ❌ GitHub Issue UI（创建/编辑/关闭/标签/指派）—— 等 v0.7.x
- ❌ GitHub 看板（与 Issue 联动）
- ❌ Gitea 任何行为变更
- ❌ OAuth2 / WebSocket 实时协作
- ❌ ListBranches GitHub 端展示（用户拍板：Git Graph 已有 branch lane，不补独立列表）
