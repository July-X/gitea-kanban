## 1. 版本定位

v0.7.1 是 v0.7.0 发版后的累积收尾版，三条主线：

1. **PR 对话区对齐 Gitea web**：用户贴 Gitea web 截图作为"期望应该长这个样子"参考，逐项对齐——评审事件拆 2 卡、合并检查警告区 toggle 行为、系统事件卡独立渲染。
2. **Timeline 数据源切换**：`/comments` 端点只返 `type=0` 普通评论；新增 `ListPullTimeline` 调 `/issues/{index}/timeline` 端点拿全 type（普通评论 + 评审 + REOPEN/CLOSE/LABEL/MILESTONE/ASSIGNEE/TITLE/REVIEW_REQUEST/MERGE/PUSH 等系统事件）。
3. **类型契约收尾**：v0.7.0 (49f0172) 引入 TimelineItemDto 重构后，shim / store / template 残留 60 个 TS 错；本版合并 5 个 commit 全部清完，`pnpm typecheck` 0 错。

> ⚠️ 本版是 typecheck + 渲染对齐收尾，**不引入新平台能力**。GitHub Issue UI、GitHub 看板、`ListBranches` GitHub 端展示、OAuth2/WS 仍按 v0.7.0 红线不做。

## 3. PR 对话区对齐 Gitea web 详情

### 3.1 评审拆 2 卡
- **旧版**：timelineItems computed 把 review.body 复制到评审事件卡内文，1 张彩色边框卡
- **新版**：服务端 `/issues/{index}/comments` 端点返 `type=21 CommentTypeReview` 的 issue comment（body 跟 review body 一致）；前端按 Gitea web 渲染 2 张独立卡（系统事件卡 + 普通评论卡）
- **根因**：v0.7.x 09a3571 调研 Gitea 1.26 源码后修的——之前 `db84089` 在 server 端 `if c.Type != 0 { continue }` 过滤所有 type，把 `type=21 (REVIEW body) / type=22 (REVIEW event) / type=1/2 (REOPEN/CLOSE) / type=28 (MERGE) / type=4 (COMMIT_REF) / type=7 (LABEL) / type=8 (MILESTONE) / type=9 (ASSIGNEE) / type=10 (TITLE) / type=27 (REVIEW_REQUEST) / type=29 (PUSH)` 等全过滤掉
- **修复**：`ListPullComments` 返所有 type，透传 type 字段给前端；前端按 type 渲染不同卡片

### 3.2 合并检查警告区 toggle
- **Wails WebView 痛点**：`<details>` 元素在系统 WebView 下二次点击不响应（4 次 revert 才落地）
- **最终方案**：`MergesView.vue` 标题行整体作为 toggle 入口——点击标题行任何地方都能展开/收起 + 收起时红框整体变矮（`collapsed` class 缩小 padding + 标题字号） + Vue 受控 toggle 配合 `@click.prevent` 拦截 WebKit default

### 3.3 系统事件卡独立渲染
- **零术语**：`timeline-item event "重新开启"` → `system_event` 卡（type=1）
- **零术语 + 中文文案 + emoji 图标**：reopen↻ / close✕ / label⚐ / milestone◐ / assignee☻ / title✎ / delete_branch⌫ / due_date⏰ / lock🔒 / change_target_branch⇄ / review_request◉ / merge⤓ / push⇧ / move◫ / dismiss_review⊘ / pin◆ / unpin◆
