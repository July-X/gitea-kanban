# v0.7.7 —— push 事件 commit 列表 + merge 事件 commit 链接

> **TL;DR**：v0.7.6 修完 5 个 user 反馈后，user 又反馈第 6 个："带分支信息的评论没正确还原" —— Gitea web 显示 push 事件带 commit 列表 + merge 事件带 base ref + SHA 链接，我们 app timeline 缺这两类详情。v0.7.7 全部修正。

## 定位

v0.7.7 是 **v0.7.4 + v0.7.5 + v0.7.6 视觉/UX 收尾后的 timeline 详情补全**：
- **v0.7.4 做过的**：DisplayName / "评论于" / header actions / 系统事件 inline 详情
- **v0.7.5 做过的**：22+ CommentType verb / 时间格式 / 移除 "事件" fallback / push 数量
- **v0.7.6 做过的**：WIP toggle / PR header / label 合并 / label 全色 / 评论 body 防御
- **v0.7.7 补的**（user 反馈 ⑥）：
  1. **push 事件 commit 列表**：缩进 + 短 SHA 链接 + commit 消息 + 提交者（对齐 Gitea web `commits_list_small.tmpl`）
  2. **push 事件 inline 块**：7 位短 SHA 链接到 Gitea web `/commit/{full_sha}` + force push 提示
  3. **merge 事件 commit 链接**：base ref + SHA 都加链接（v0.7.6 是纯文本）

## 范围

### 后端（1 个 commit）

  - `app/platform/adapter.go: TimelineItem` 加 5 字段：

### 前端（1 个 commit）

- **`feat(v0.7.7): 前端 push 事件 commit 列表 + merge 事件 commit 链接`**
  - **DTO 同步**：`TimelineItemDto` 加 5 字段
  - **commitsByPR 缓存**：`Map<index, PullCommitDto[]>` 缓存 PR 全量 commit 列表
    - `loadCommits(projectId, index)` helper：缓存命中直接返
    - `loadComments` 调完 `fetchTimeline` 后并行 `loadCommits`（push 事件渲染用）
  - **push 事件 inline 块**：
    - 7 位短 SHA 链接到 Gitea web `/commit/{full_sha}`（Gitea / GitHub 通用）
    - force push 时加 "(强制推送)" 提示
  - **push 事件 block 块**（独立于 `hasSystemEventBlockDetail` 链）：
    - 缩进 22px + 左侧 2px 分隔线（对齐 Gitea web `commits_list_small`）
    - 每行：`GitCommit` icon + 短 SHA 链接 + commit 消息 + 提交者
  - **merge 事件 inline 块**：
    - SHA 改成 `<a>` 链接到 Gitea web `/commit/{sha}`
    - 优先用 `item.mergeCommitSha`（v0.7.7 新加字段）
    - fallback 用 body regex 抠的 7 位短码（兼容 v0.7.6 老数据）
  - **新 helpers**：
    - `fullMergeSha(item)`：从 body regex 抠完整 40 位 SHA（`mergeCommitSha` 只返 7 位）
    - `pushEventCommits(item)`：按 NewCommit 位置 + CommitsNum 从 commitsByPR 过滤

  - **CSS 8 处新增**：`.pr-detail__event-block--commits` / `.pr-detail__event-commit-row` / 4 个子元素样式

## 已知 limitation

- **push 事件 commit 列表分组**：
  - Gitea 端 `Comment.Commits` 数组是 `xorm:"-"` 服务端字段，API 不暴露
  - v0.7.7 简化版：按 `NewCommit` 位置 + `CommitsNum` 从 `commitsByPR` 过滤（可能跟 Gitea web 实际分组不一致，比如 force push 后 NewCommit 位置变了）
  - v0.7.7.1 计划：用 Gitea `/compare/{OldCommit}...{NewCommit}` 端点拿精确 commit 范围
- **force push compare 链接**：Gitea web 在 force push 时显示 compare 链接，v0.7.7 暂没渲染（只加 "(强制推送)" 提示），v0.7.7.1 补
- **commitsByPR 缓存**：
  - 当前按 PR index 缓存（用户切到别的 PR 不会清空，保留上下文）
  - 大 PR（>500 commits）可能占较多内存，v0.7.8 考虑 LRU + 限大小
