# v0.8.18 — 设置页 Git 二进制 radio 单选 + gh 信息修复

> 发布日期：2026-07-22
> commit：`4f2a80c`

## 问题背景

1. **Git 二进制卡片 UI 不一致**：外观主题和 gh 二进制卡片已改为 radio 单选样式，但 Git 二进制卡片仍使用旧的 toggle button 分组，视觉风格不统一。
2. **gh 信息一直「加载中…」**：`ipc-client.ts` 的 `getGhBinaryConfig()` 调用 `invoke('ghBinary', 'getConfig', {})`，但 `wails-api-shim.ts` 里缺失 `ghBinary` namespace，导致 Promise 永远 pending。
3. **gh 独立卡片冗余**：gh 只是 GitHub 平台的辅助提示，不需要独立的设置卡片；内嵌 Git 已能覆盖所有场景。

## 改动内容

### 1. Git 二进制卡片 → radio 单选

**文件**：`frontend/src/views/SettingsView.vue`

- 移除 `settings__mode-toggle` + `button.settings__mode-btn` 旧样式
- 替换为 `settings__theme-opt` + `input[type=radio]`，与外观主题 / gh 二进制卡片样式统一
- 默认值从 `'custom'` 改为 `'embedded'`，启动期默认选中「使用内嵌 Git」

### 2. 修复 ghBinary namespace 缺失

**文件**：`frontend/src/lib/wails-api-shim.ts`

- 在 `WailsApp` 类型中添加 `GetGhBinaryConfig?` / `SetGhBinaryPath?` / `TestGhBinary?` 三个可选字段
- 在 `gitBinary` 和 `system` 之间新增 `ghBinary` 子 namespace，包含 `getConfig` / `setPath` / `test` 三个方法
- 走 `forwardToWails` 转发到 Go 后端 Wails bindings

### 3. 移除 gh 独立卡片

**文件**：`frontend/src/views/SettingsView.vue`

- 删除「gh 二进制」独立卡片模板（`v-if="showGhBinaryCard"` 的整个 section）
- 删除 script 中 6 个响应式状态 + 3 个函数（`ghBinaryConfig` / `ghBinaryLoading` / `ghBinaryTestResult` / `ghBinaryMode` / `ghBinaryEffectiveVersion` / `showGhBinaryCard` / `loadGhBinaryConfig` / `onSelectGhSystem` / `onPickAndUseCustomGh`）
- 清理 import 中的 `getGhBinaryConfig` / `setGhBinaryPath` / `testGhBinary` / `GhBinaryResult` / `TestGhResult`
- `onMounted` 移除 `loadGhBinaryConfig()` 调用

### 4. 精简 toast 提示

- `onSelectEmbedded` 去掉成功分支 toast，只保留 warn 分支（内嵌 git 不能跑时提示）
- `onPickAndUseSystemGit` 去掉成功分支 + 失败分支 toast，只保留文件选择器打开失败 toast
- radio 模式下状态行已承载信息反馈，无需额外 toast
