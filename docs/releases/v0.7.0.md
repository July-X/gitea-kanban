## 1. 版本定位

v0.7.0 把 GitHub 平台能力补齐到「和 Gitea 对等可用」—— GitHub PR 详情页的属性编辑器（标签 / 指派人 / 评审人 / 里程碑）从 UI 隐藏升级为可读写。

**核心要点**：
- **GitHub adapter 后端补全 5 个方法**：ListLabels / ListMembers / ListMilestones / UpdatePullMilestone / ListPullCommits（Phase 1，1.5 周）
- **属性编辑器对 GitHub 数据源可用**：打开 v-if + 派发 milestone state（gitea='all' / github='open'）+ 修 v0.6.0 潜伏 store action bug（Phase 2，1.5 周）
- **GitHub milestone 进入 PR 详情显示**：GetPull 增加 Milestone 映射（number → DTO.ID）
- **CreatePullReview 支持行内评论**：原硬编码 `comments: []` 改透传 `opts.Comments`
- **跨平台 build CI**：macOS universal / Windows / Linux 三个并行 build jobs

> ⚠️ v0.7.0 不做：GitHub Issue UI、GitHub 看板、ListBranches GitHub 端展示、EB 检测 /ahead-behind 提示（应用只拉元数据）。

## 4. 关键设计决策

### D1：UpdatePullMilestone GitHub 用 number vs Gitea 用 name

| 维度 | Gitea | GitHub |
|---|---|---|
| API 字段 | `milestone: "v1.0"` (title) | `milestone: 1` (number) |
| 响应 | `"milestone": "v1.0"` | `"milestone": {number: 1, title: "v1.0"}` |
| 前端 store | 传 title | 传 title |

**决策**：DTO 字段类型不变（`MilestoneDTO.Title`），GitHub adapter 内部做 title→number 反查（ListMilestones 预加载建立 map）。前端 store 仍按 name 显示。

`UpdatePullMilestoneArgs` 字段名/类型不变，新增 GitHub title→number 转换在 adapter 内部。

### D2：ListBranches GitHub 数据源下不展示分支列表（用户拍板）

**决策**：
- GitHub `ListBranches` 仍返 `ErrNotSupported`（保持 v0.6.0 现状）
- 前端 `branch store` 在 GitHub 数据源下跳过加载（store action 内部 `if (platform === 'github') return []`）
- SettingsView 入口在 GitHub 数据源下隐藏

**理由**：Git Graph 已有 branch lane 渲染 + branch tip dot，UI 已覆盖；避免 N×API 调 + monorepo rate limit 风险。

### D3：Phase 2 必修 v0.6.0 潜伏 store action 缺失 bug

v0.6.0 Phase 3 commit `61b1464` 在 plan 里 claim 加了 `loadAttrEditorData` / `updateLabels` / `updateAssignees` / `updateReviewers` / `updateMilestone` 5 个 store action，但实际上**只 import 了 IPC bindings，没有真加到 store**。MergesView.saveAttrs 4 个动作全是空调用，被 try/catch 静默吞掉。pnpm typecheck 报 8 条 TS2339，被视为预存错误忽略。

**修复**：commit `d0957b2` 真的加了 6 个 store action + state refs + 乐观更新。
