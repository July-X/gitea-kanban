# v0.7.2 —— 视觉 1:1 对齐 Gitea web（PR 对话区）

> **TL;DR**：v0.7.1 的 "对齐 Gitea web" 实际只做了结构层（3 路分支 + 评审拆 2 卡），
> 视觉层差距大。v0.7.2 补齐 5 档颜色、lucide 图标体系、7 类系统事件二级详情块、
> 气泡左箭头、Dismiss review 拆 2 卡 —— 整体视觉与 Gitea web 接近 1:1。

## 定位

接续 v0.7.1 "PR 对话区结构对齐 Gitea web"，v0.7.2 把视觉层 / 细节层做实：

- **v0.7.1 做过的**（结构层）：3 路分支（comment / review / system event）、评审拆 2 卡（event + body）、合并检查警告区 toggle、系统事件卡独立渲染、Timeline 数据源切换到 `/issues/{index}/timeline`
- **v0.7.2 补的**（视觉层）：
  - 5 档颜色（success / danger / merge / warn / neutral）—— Gitea web 用语义色编码事件
  - 21 个 lucide 图标替代 Unicode 字符 —— 对齐 Gitea web octicon-* 体系
  - 7 类系统事件二级详情块（label / milestone / assignees / title / branch / ref issue / dependency）
  - 评论气泡左箭头 CSS 三角形 —— 对齐 Gitea web `.avatar-content-left-arrow`
  - Dismiss review 拆 2 卡（event + reason comment 卡）—— Gitea web Type=32 在 body 非空时拆

### 仍然做不了的部分（API 限制）

- **Type=29 Push commit 列表**：Gitea `models/issues/comment.go` 的 `Commits []*git_model.SignCommitWithStatuses` 是 `xorm:"-"` 字段，**API 端点不暴露**。Gitea web 用服务端模板直渲染，我们走 JSON API 拿不到。v0.7.2 只能显示 "推送了新提交" 通用文案，没有 commit 列表。
- **Type=27 review_request 详细评审人**：Gitea `api.Issue` 没暴露 `RequestedReviewer` 列表，只有 `Assignee` + `RemovedAssignee` 标志。v0.7.2 评审请求事件显示 "请求评审" + assignee，完整评审人列表走 PR 详情页面的"评审人"区域。
- **Type=36/37 pin/unpin 详情**：无 detail 字段要渲染，保持纯文本。
