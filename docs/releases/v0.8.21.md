# v0.8.21 — 安装版（macOS dmg）暗色滚动条白底修复

> 发布日期：2026-07-23
> 关联 commit：da1ac035（v0.8.19 滚动条修复起点）+ 全局 color-scheme 引入

## 问题背景

User 反馈：在 macOS 安装版（gitea-kanban-vX.Y.Z-macos-amd64.dmg 安装后的 .app）暗色模式下，滚动条呈现"白底"，与 dev 模式（wails dev）下观感不一致。

v0.8.19（commit `06b7196`，包含 `da1ac035` 的滚动条修复部分）只在 3 个具体组件（CommitDetailPanel + MergesView.comment-list + TimelineNewView.commit-accordion）显式设了 `::-webkit-scrollbar-track` 走 `--scrollbar-track` token，其他 30+ 个 `overflow-y: auto` 容器依赖全局兜底。

但 **macOS WKWebView 在 overlay scrollbar mode 下，没有显式 `color-scheme: dark`** 时，会按 system 当前 color-scheme 渲染 scrollbar 容器背景：用户系统是暗色但应用 CSS 没声明 `color-scheme: dark` → WKWebView 仍 fallback 到 system **light** 滚动条底色——所以即使 `::-webkit-scrollbar-track` 走 `var(--scrollbar-track)`（4% alpha 冷白），在 WKWebView overlay scrollbar 层外侧的轨道容器仍呈现系统浅色，肉眼看就是"白底"。

dev 模式因为 Vite + 系统 WebView 与 WKWebView 行为差异，恰好在某些情况下不暴露此问题。

## 改动内容

### 1. theme.css 加全局 color-scheme

**文件**：frontend/src/styles/theme.css

```css
:root { color-scheme: dark; }
html[data-theme='light'] { color-scheme: light; }
@media (prefers-color-scheme: light) { :root:not([data-theme='dark']) { color-scheme: light; } }
```

效果：WKWebView 在 dark / light 应用主题下，按应用主题渲染 overlay scrollbar 容器底色，不再 fallback 到 system color-scheme。这是 v0.9.x 安装版滚动条修复的关键性兜底。

### 2. TimelineNewView.commit-accordion::-webkit-scrollbar-track transparent 改走 token

**文件**：frontend/src/views/TimelineNewView.vue

`background: transparent` → `background: var(--scrollbar-track)`。v0.8.19 当时没扫到这个组件。

### 3. 重复声明清理（DRY）

**文件**：CommitDetailPanel.vue + MergesView.vue + TimelineNewView.vue

三个组件重复声明 `::-webkit-scrollbar-track/thumb` 规则——全局兜底在 theme.css 已经定义，组件层只保留组件特有的 `width: 6px` 等差异。CommitDetailPanel 的 `scrollbar-color` 从 `transparent` 改成 `var(--scrollbar-track)` 走主题 token。
