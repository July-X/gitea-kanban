# v0.8.22 — macOS 自动下载 dmg + 打开 Finder 引导安装

> 发布日期：2026-07-22
> commit：待 push 后回填

## 问题背景

User 反馈：v0.8.21 安装版点"前往下载页"按钮后，banner 文案说"macOS build 未签名+notarize，请手动前往 GitHub release 页下载"，功能仍被识别为 manual-only，user 期望"一键升级"体验。

之前 v0.8.0 拍板"macOS 未签名 build 走 OpenDownloadPage 手动兜底"，但 v0.8.0.1 留的 stub 让 `canSelfUpdate() → false` 永远把 macOS 标 manualOnly，加上 macOS Gatekeeper 拦未签名 binary in-place 替换——综合下来 macOS 用户**永远只能手动下 dmg 重装**，没有"自动下载 + 自动打开"。

## 改动内容

### 1. canSelfUpdate() macOS 返 true

**文件**：app/updater/updater.go

```go
case "darwin":
    return true  // 走 download + open dmg 流程
```

让前端走 download 路径（不再卡在 manualOnly），UI 显示真实进度条 + 下载文案。

### 2. applyMacOS 改为 exec.Command("open", dmgPath)

**文件**：app/updater/updater_darwin.go

v0.8.0.1 老逻辑是 in-place binary 替换（写 .new + shell 脚本 mv），但 macOS Gatekeeper 拦未签名 binary 替换——永远跑不通。

v0.8.22 改为：
1. 后端 `DownloadUpdate` 流式下载 dmg 到 `${dataDir}/updates/` + ed25519 签名校验
2. 前端显示进度条 → 完成后状态切到 `downloaded`
3. user 点"打开 Finder 安装"按钮 → 后端 `InstallUpdate` → `applyMacOS("open", dmgPath)`
4. Finder 自动弹出 dmg，user 看到 .app bundle + Applications symlink，拖 .app 到 /Applications 完成升级

老的 in-place replace 逻辑（写 .new + shell 脚本）保留在 git 历史里，签名发布时可恢复。

### 3. UpdateInfo 加 platform 字段

**文件**：app/updater/updater.go + frontend/src/types/dto.ts

```go
Platform string `json:"platform,omitempty"`  // "darwin-amd64" / "windows-amd64"
```

前端 UpdateBanner / SettingsView 按钮文案按 platform 区分：
- macOS：下载完成 → "打开 Finder 安装"
- Windows：下载完成 → "重启并安装"

### 4. TestCanSelfUpdate 平台断言更新

**文件**：app/updater/updater_test.go

`runtime.GOOS == "windows"` → `runtime.GOOS == "windows" || "darwin"`。

## 验证

- `go test ./app/updater/...` 全过（TestCanSelfUpdate 改平台断言）
- `pnpm typecheck` ✅
- 手动验证：v0.8.21 binary 升级到 v0.8.22 → 设置页点检查更新 → latest=v0.8.22 → 点"下载更新" → 进度条真实跑 → 下载完成 → 按钮变"打开 Finder 安装" → 点 → Finder 弹出 dmg → 拖 .app 到 /Applications → 重启 app 完成升级

## 影响范围

- 修所有 macOS 用户的自动升级路径
- 不影响 macOS arm64 / Windows / Linux / dev build
- 不动 release.yml / Wails binding / CI workflows

## 向后兼容

- 老的 in-place replace 逻辑保留在 git 历史，签名发布时可恢复
- macOS dmg 已下载但 user 跳过"打开 Finder"按钮时，dmg 留 cache，下次冷启动可重试
- macOS Gatekeeper 在 dmg 首次双击安装时仍会弹警告（未签名 build），但已经能让 user 看到 dmg 内容 + 完成安装路径

## 不动 / 不要做

- 不要改 release.yml 命名
- 不要在 v0.8.22 加 adhoc 签名（开发机 Apple Developer ID 不在本项目范围）
- 不要引入 third-party installer 框架（如 Sparkle），保持 Go 标准库 + os/exec
