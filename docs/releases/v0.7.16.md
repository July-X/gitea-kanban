# v0.7.16 —— merge 事件整段 white-space: nowrap 强制 1 行

> **TL;DR**：user 反馈"合并提交 X 到 Y"应该 1 行显示。当前 master 跑出来是 2 行（"X 于 Y 合并提交" 主行 + 短码 "X 到 Y" 下一行），根因是 v0.7.10 加的 `flex-wrap: wrap` 让长内容在主行宽度不够时换行。v0.7.16 给 merge 事件整段加 `white-space: nowrap` 强制 1 行，对齐 Gitea web。

## 范围

### 前端（1 个 commit，master 直接改）

- **`fix(v0.7.16): merge 事件整段 white-space: nowrap 强制 1 行渲染`** —— frontend/src/views/MergesView.vue

## 改动详情

### 实际效果对比（merge 事件 timeline）

| 改前 | 改后 | Gitea web 对照 |
|---|---|---|
| 主行: "kanban_bot 于 27 天前 合并提交" | 主行: "kanban_bot 于 27 天前 合并提交 7db04cd 到 main" | "kanban_bot 于 27 天前 合并提交 7db04cd 到 main" |
| 下一行: "7db04cd 到 main" | （无下一行） | （无下一行） |

### 改动 2 处（1 个 CSS + 1 个 template）

1. **新增 CSS `.pr-detail__event-merge`**：

```css
.pr-detail__event-merge {
  white-space: nowrap;     /* 强制 1 行 */
  display: inline-flex;
  align-items: center;
  gap: 4px;
}
```

2. **merge 事件 v-else-if 链 span 加 class**：

```vue
<span v-else-if="item.type === 'merge' && selectedPR?.mergeCommitSha && selectedPR?.base?.ref"
      class="pr-detail__event-merge">
  <a>{{ ShortSha }}</a>
  <span>到</span>
  <code>{{ baseLabel(selectedPR) }}</code>
</span>
```

## 根因

v0.7.10 我加的 `pr-detail__event-line` CSS `flex-wrap: wrap` 让主行内容多时换行。当时是为了支持 author / time 等太长时换行不撑爆容器。merge 事件整段（verb "合并提交" + ShortSha + 到 + branch）有 7 个 inline element，主行宽度不够时**整段换行**到下一行——但 user 期望跟 Gitea web 一致 1 行渲染。

## 修复

只 merge 事件整段加 `white-space: nowrap`，主行 `flex-wrap: wrap` 保留（author / time 等太长仍可换行）。
