# v0.7.3 —— Timeline 视觉对齐 Gitea web（系统事件紧凑单行 + 左侧竖线）

> **TL;DR**：v0.7.2 的系统事件套了跟评论一样的 flex layout + bubble 框，看上去又大又重。Gitea web 截图对比显示 system event 是**单行紧凑布局** + 左侧贯穿**灰色 timeline 竖线** + 圆点/icon 节点把竖线"切断"在节点处。v0.7.3 修复这个视觉层问题。

## 定位

接续 v0.7.2 "PR 对话区视觉 1:1 对齐 Gitea web"，v0.7.3 把 timeline 视觉结构做实：

- **v0.7.2 做过的**（静态视觉）：5 档颜色、21 个 lucide icon、7 类系统事件二级详情块、bubble 左箭头、Dismiss 拆 2 卡
- **v0.7.3 补的**（动态视觉）：
  - system event 紧凑单行布局 —— 去掉 bubble 框，纯 icon + 单行文字
  - 左侧贯穿灰色 timeline 竖线（2px，CSS 三角形节点切断）
  - 5 档颜色应用到 dot 边框（不只是文字色）
  - 评审 state 独立 dot 颜色
  - 二级详情拆 inline / block 两层（label chip inline，ref issue 链接换行）

## 范围

### 前端（1 个 commit）

- **`feat(v0.7.3): 系统事件紧凑单行 + 左侧 timeline 竖线 + dot 节点`**
  - 模板：`<ul class="pr-detail__comment-list">` → `<ul class="pr-detail__timeline">`
  - 4 个分支结构大改：
    | 类型 | 布局 |
    |------|------|
    | `--comment` | 左 avatar（圆形 + 2px 边框切竖线）+ 右 bubble（保持原 header + body + actions）|
    | `--event --review` | 左 dot（按 review state 上色：approved=绿 / changes_requested=红 / commented=灰）+ 右单行（用户名 + 评审状态 + 时间）|
    | `--event --system` | 左 dot（按 system color 上色：5 档）+ 右 event-line（作者 + 事件 + 时间）+ 可选 event-inline（label chip / milestone / branch 等小信息）+ 可选 event-block（ref issue / dependency 链接 + 标题）|
    | dismiss reason | 复用 `--comment` 结构（独立 timeline item，按 comment 卡渲染）|
  - avatar / dot 节点用 `pr-detail__timeline-rail` 容器 + `position: absolute; left: -32px` 定位到 timeline 竖线上
  - dot 用 `pr-detail__timeline-dot` 圆形 + 1.5px 边框，按 system color 决定边框色
  - 5 档颜色（success / danger / merge / warn / neutral）应用到 dot 边框 + icon 色
  - 评审 state 独立 dot 颜色（绿 / 红 / 灰）
  - 二级详情拆 inline + block 两层（v0.7.2 是单一 `.comment-event-detail` 混合）
  - CSS：
    - `.pr-detail__timeline::before` 画左侧 2px 灰色竖线（top: 14px; bottom: 14px; left: 14px）
    - `.pr-detail__timeline-rail` 绝对定位 -32px，z-index: 1 切断竖线
    - `.pr-detail__timeline-avatar` / `.pr-detail__timeline-dot` 圆形节点
    - `.pr-detail__event-line` / `.pr-detail__event-inline` / `.pr-detail__event-block` 三层结构
  - `hasSystemEventDetail` 拆成 `hasSystemEventInlineDetail` + `hasSystemEventBlockDetail` 两个函数

### 删除的旧样式

- `.pr-detail__comment-side` / `.pr-detail__comment-name`（被 pr-detail__timeline-rail + comment-meta 替代）
- `.pr-detail__comment--system-event` 整体块（被 pr-detail__timeline-item--event --system 替代）
- `.pr-detail__comment--system-color-*`（被 pr-detail__timeline-dot--* 替代）
- `.pr-detail__comment-bubble--event`（不用了）
- `.pr-detail__comment-avatar--approved/--changes_requested/--commented`（被 timeline-dot--review-* 替代）
- `.pr-detail__comment--review-event` 虚线边框（被 dot 颜色替代）
